# Type: 50자 이내로 명료하게 작성
# -- list --
#   - Feat      → 새로운 기능 추가
#   - Fix       → 버그 수정
#   - Docs      → 문서 수정
#   - Style     → 코드 포맷팅, ';' 누락, 기타 기능 외 수정
#   - Refactor  → 기능 수정이 없는 리펙토링
#   - Test      → 테스트 관련 모든 추가
#   - Init      → 프로젝트 생성
#   - Rename    → 파일의 이름, 오타만 수정
#   - Remove    → 파일 삭제만 하는 경우
#   - Chore     → 애매한 부분
# (Part)Type: Subject 
# 위 형식으로 작성
#   - Type은 첫 글자 대문자
#   - Part는 AD, Backend, Frontend
#       - 첫 글자 대문자
#       - (Part)이후 띄어쓰기 없이 ':'
#   - Subject는 제목을 50자 이내로 설명
#       - ':' 뒤 띄어쓰기
#       - 첫 글자 대문자
#       - 개조식 구문으로 작성
#       - 제목 끝에 (.)은 없도록 작성
# ex) (Backend)Feat: Modify DB

# Body: 
# type으로 불충분한 경우 설명 작성
# 필요 없는 경우는 Body 생략
# ex) Body: A 클래스를 B와 C 클래스로 분리
#       - B : D 기능 수행
#       - C : E 기능 수행

# Footer:
# -- list --
#   - Fixes         → 이슈 수정 중(해결 된 경우는 Resolves)
#   - Resolves      → 이슈 해결했을 때 사용
#   - Ref           → 참고할 이슈가 있을 때 사용(참고할 이슈는 Resolves 상태)
#   - Related to    → 해당 커밋과 직/간접적으로 관련된 이슈번호
# ex) Fixes: #53 Related to: #13, #42
